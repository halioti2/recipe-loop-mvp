# CLAUDE.md

## Documentation Standards

### Research Documents

**IMPORTANT:** Research documents are created on-demand only when explicitly requested. Do not proactively create research documents - wait for the user to specifically ask for "architecture research" or similar.

#### Architecture Research
- **Naming:** `docs/research/RE_[topic].md` — date goes inside the document, not in the filename
- **Purpose:** Analyze design patterns, feasibility, trade-offs, and best practices
- **Required Sections:**
  - Executive Summary (key findings/TL;DR)
  - Sequence Diagram (interactions between systems)
  - Architecture Diagram (data flows, system architecture)
  - Data Flow Diagram (decision trees, process flows, state transitions)
  - Problem vs Solution (comparison table/matrix)
  - Real-World Examples (with links and analysis)
  - Side-by-Side Comparison (code/config/approach comparison)
  - Sources (ONLY if websearch was used, with executive summary of each)
- **Template & Examples:** See `docs/research/TEMPLATE.md`
